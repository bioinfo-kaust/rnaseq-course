Lab 8: Functional Enrichment Analysis

In this lab, you will take the differentially expressed genes identified in Lab 7 (TDP-43/TARDBP knockdown vs. control, GSE136366) and ask: what biological processes and pathways are dysregulated? You will use two complementary approaches — Over-Representation Analysis (ORA) and Gene Set Enrichment Analysis (GSEA) — implemented in R using clusterProfiler and fgsea.

Working with chr11-only results

The DESeq2 results from Lab 7 cover only chromosome 11 genes. Gene set databases (GO, KEGG, MSigDB Hallmarks) span the entire genome, so enrichment analysis may return fewer significant terms than a full-genome run. This is expected — the methodology and interpretation are identical. For course purposes, the chr11 results are sufficient to demonstrate the workflow.

Part 2: Prepare Gene Lists

Enrichment methods require gene identifiers in specific formats. KEGG and many gene set databases use Entrez IDs, while your DESeq2 results use Ensembl IDs. This section loads the results and performs ID conversion.

Load DESeq2 results

# Load the differential expression results from Lab 7
res_df <- read.csv("results/DESeq2_TDP43KD_vs_control.csv")

# Inspect the data
head(res_df)
dim(res_df)
colnames(res_df)
Expected columns in DESeq2 results

The file should contain at minimum: gene_id (Ensembl IDs), log2FoldChange, pvalue, padj, and stat (the Wald test statistic). If your column names differ, adjust the code below accordingly.

Convert Ensembl IDs to Entrez IDs

KEGG enrichment requires Entrez gene IDs. Use the org.Hs.eg.db annotation package to map Ensembl IDs to Entrez IDs.

# Map Ensembl gene IDs to Entrez IDs
res_df$entrez_id <- mapIds(org.Hs.eg.db,
    keys      = res_df$gene_id,
    column    = "ENTREZID",
    keytype   = "ENSEMBL",
    multiVals = "first")

# Remove genes that could not be mapped
res_df <- res_df %>% filter(!is.na(entrez_id))

# Check how many genes remain
nrow(res_df)
Why do some genes fail to map?

Not all Ensembl IDs have a corresponding Entrez ID. This typically affects pseudogenes, non-coding RNA genes, and genes on alternative contigs. Losing a small fraction (~5-10%) is expected and acceptable.

Define significant gene lists for ORA

# All significant DEGs (padj < 0.05, |LFC| > 1) — used for ORA
sig_genes <- res_df %>%
    filter(padj < 0.05, abs(log2FoldChange) > 1) %>%
    pull(entrez_id)

# Upregulated genes only
up_genes <- res_df %>%
    filter(padj < 0.05, log2FoldChange > 1) %>%
    pull(entrez_id)

# Downregulated genes only
down_genes <- res_df %>%
    filter(padj < 0.05, log2FoldChange < -1) %>%
    pull(entrez_id)

# Universe = all genes that were tested (not just significant ones)
universe_genes <- res_df$entrez_id

cat("Significant DEGs:", length(sig_genes), "\n")
cat("Upregulated:", length(up_genes), "\n")
cat("Downregulated:", length(down_genes), "\n")
cat("Universe (all tested):", length(universe_genes), "\n")
The gene universe matters — always define it explicitly

In ORA, statistical significance is calculated as the probability of observing this many hits in a gene set by chance, given how many genes you tested overall. If you do not specify the universe (background), the tool defaults to all genes in the database — this inflates the denominator and produces misleading p-values. Always set the universe to the genes that were actually present and testable in your experiment (i.e., all genes that passed count filters in DESeq2).

Part 3: GO Enrichment (Over-Representation Analysis)

What is Over-Representation Analysis (ORA)?

ORA asks: given a list of significant genes, are certain GO terms (or pathways) represented more than expected by chance? It uses a hypergeometric test (similar to Fisher's exact test) that counts how many of your significant genes belong to each GO term, compared to how many would be expected given the term's size in the background. ORA is intuitive but has a key limitation: it discards all quantitative information (fold-changes, p-values) by applying a hard significance cutoff to define the gene list.

When to use ORA: When you have a clearly defined list of "hits" and want a fast, interpretable summary of their biology.

Run GO Biological Process enrichment

# GO Biological Process enrichment
ego_BP <- enrichGO(
    gene          = sig_genes,
    universe      = universe_genes,
    OrgDb         = org.Hs.eg.db,
    ont           = "BP",        # Biological Process
    pAdjustMethod = "BH",        # Benjamini-Hochberg correction
    pvalueCutoff  = 0.05,
    qvalueCutoff  = 0.2,
    readable      = TRUE         # convert Entrez IDs back to gene symbols
)

# Inspect top results
head(ego_BP, 10)

Tip: Interpreting the three GO ontologies
  • Biological Process (BP) — What process is the gene involved in? (e.g., mRNA splicing, apoptosis). Most informative for RNA-seq.
  • Molecular Function (MF) — What molecular activity does the gene have? (e.g., RNA binding, kinase activity). Useful for identifying protein function classes.
  • Cellular Component (CC) — Where in the cell does the gene product act? (e.g., nucleus, spliceosome). Useful for subcellular context.

For TDP-43 (an RNA-binding protein), expect BP enrichment in RNA splicing and processing, MF enrichment in RNA binding, and CC enrichment in the nucleus and ribonucleoprotein complexes.

Part 4: KEGG Pathway Enrichment

GO terms vs. KEGG pathways — what is the difference?

Gene Ontology (GO) terms are hierarchical, ontology-based annotations that describe molecular functions, biological processes, and cellular components. They are broad and cover virtually every gene in the genome.

KEGG (Kyoto Encyclopedia of Genes and Genomes) pathways are manually curated metabolic and signaling pathway maps. They are more focused on molecular interaction networks and metabolic reactions, and are particularly useful when you want to understand how dysregulated genes fit into known biochemical pathways (e.g., the TCA cycle, MAPK signaling). KEGG pathways are fewer in number than GO terms but tend to be more interpretable as coherent biological units.

Run KEGG pathway enrichment

# KEGG pathway enrichment (requires Entrez IDs and internet access)

# Inspect top results

Tip: KEGG organism codes

KEGG uses three-letter organism codes. Common ones: hsa (human), mmu (mouse), rno (rat), dre (zebrafish). Find codes at kegg.jp/kegg/catalog/org_list.html.

Part 5: Gene Set Enrichment Analysis (GSEA)

What is GSEA and how does it differ from ORA?

GSEA (Gene Set Enrichment Analysis) does not require a hard significance cutoff. Instead, it uses all genes ranked by a continuous score (typically the Wald statistic or sign(LFC) * -log10(p-value)). It then tests whether the genes in a predefined gene set are non-randomly distributed toward the top or bottom of the ranked list — indicating coordinated upregulation or downregulation.

Key advantages over ORA:

  • Uses all genes, not just a binary significant/not-significant split — more statistical power
  • Can detect subtle but coordinated shifts in gene sets that no individual gene reaches significance
  • The direction of enrichment (positive or negative NES) tells you whether the gene set is overall up- or down-regulated

When to use GSEA: When you want to detect coordinated pathway-level changes, especially when individual gene-level signals are modest (e.g., in complex diseases or subtle perturbations).

Build a ranked gene list

Rank all tested genes by their DESeq2 Wald statistic. Positive values indicate upregulation in TDP-43 KD; negative values indicate downregulation.

# Build ranked list: all genes sorted by Wald statistic (descending)
# Note: use .data$stat to avoid conflict with the stats::stat function
ranked_list <- res_df %>%
    filter(!is.na(.data$stat), !is.na(entrez_id)) %>%
    arrange(desc(.data$stat)) %>%
    select(entrez_id, stat)

# Create a named numeric vector (required by fgsea)
gene_ranks <- ranked_list$stat
names(gene_ranks) <- ranked_list$entrez_id

# Remove duplicate Entrez IDs (keep first occurrence, which has highest |stat|)
gene_ranks <- gene_ranks[!duplicated(names(gene_ranks))]

cat("Number of genes in ranked list:", length(gene_ranks), "\n")
# Expected: ~15,000 to 20,000 genes

# Inspect the top and bottom of the ranked list
head(gene_ranks, 5)    # most upregulated
tail(gene_ranks, 5)    # most downregulated

Download Hallmark gene sets from MSigDB

# MSigDB Hallmark collection: 50 curated gene sets representing well-defined biological states
msig_hallmark <- msigdbr(species="Homo sapiens", category="H")

# Convert to a named list of Entrez ID vectors (format required by fgsea)
hallmark_sets <- split(msig_hallmark$entrez_gene, msig_hallmark$gs_name)

# How many gene sets?
length(hallmark_sets)

# Preview one gene set
head(hallmark_sets[["HALLMARK_MYC_TARGETS_V1"]], 10)
Tip: MSigDB collections

MSigDB provides several gene set collections. For RNA-seq, the most commonly used are:

  • H (Hallmark) — 50 well-defined biological states; low redundancy, high interpretability. Best starting point.
  • C2 (Curated) — Reactome, KEGG, and other curated pathway gene sets.
  • C5 (Ontology) — GO gene sets. Large but redundant.
  • C6 (Oncogenic) — Cancer-relevant signatures.

Use msigdbr_collections() to see all available collections.

Run fgsea

# Run fgsea (fast GSEA implementation)
set.seed(42)   # for reproducibility
fgsea_results <- fgsea(
    pathways    = hallmark_sets,
    stats       = gene_ranks,
    minSize     = 15,          # minimum genes in a set to test it
    maxSize     = 500,         # maximum genes in a set
    nPermSimple = 1000         # number of permutations
)

# Sort by NES and filter to significant results
fgsea_results <- fgsea_results %>%
    arrange(desc(NES)) %>%
    filter(padj < 0.05)

# Show top results
head(fgsea_results[, c("pathway", "NES", "padj", "size")], 15)
Understanding the NES (Normalized Enrichment Score)

The Enrichment Score (ES) measures the degree to which genes in a gene set are overrepresented at the top or bottom of the ranked gene list. The Normalized Enrichment Score (NES) corrects the ES for gene set size, making scores comparable across gene sets of different sizes.

  • Positive NES: Gene set genes are concentrated at the top of the ranked list — the pathway is overall upregulated in TDP-43 KD relative to control.
  • Negative NES: Gene set genes are concentrated at the bottom — the pathway is overall downregulated.
  • NES near 0: No enrichment signal; genes are randomly distributed across the list.

Use padj (BH-corrected p-value across all tested gene sets) to determine significance. The NES magnitude tells you about the strength and direction of enrichment.

Visualize GSEA results — barplot of all significant pathways

# Prepare data for plotting

Plot a single pathway enrichment curve

The enrichment plot shows the running enrichment score across the ranked gene list, and the distribution of gene set members (black tick marks at the bottom).

# Identify the most enriched pathway (highest NES)
top_pathway <- fgsea_results$pathway[which.max(fgsea_results$NES)]
cat("Top enriched pathway:", top_pathway, "\n")

# Plot the enrichment curve for that pathway
plotEnrichment(hallmark_sets[[top_pathway]], gene_ranks) +
    labs(title=top_pathway,
         subtitle=paste0("NES = ", round(fgsea_results$NES[fgsea_results$pathway == top_pathway], 3),
                         "  padj = ", signif(fgsea_results$padj[fgsea_results$pathway == top_pathway], 3)))
Tip: Reading the enrichment plot

The green running-sum line rises steeply when it encounters members of the gene set. The peak of this curve is the enrichment score (ES). The black tick marks along the x-axis show where in the ranked gene list each gene set member falls. A peak on the left means genes cluster toward the top (upregulated); a peak on the right means they cluster toward the bottom (downregulated).
