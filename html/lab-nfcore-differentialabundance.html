Lab 9: Standardized Differential Abundance Analysis with nf-core/differentialabundance

Part 1Setup Part 2Prepare Inputs Part 3Run Pipeline Part 4Output Structure Part 5Shiny Report Part 6MultiQC Report

In this lab, you will run the nf-core/differentialabundance workflow on the GSE136366 (TDP-43/TARDBP knockdown) dataset using Salmon quantification output produced in the nf-core/rnaseq chr11 run (Lab 6). Because the chr11-filtered Salmon output is used, the analysis covers only chromosome 11 genes — the pipeline steps and output structure are identical to a full-genome run. You will prepare the required input files, execute the pipeline on Ibex, and interpret the resulting DESeq2 output tables and interactive Shiny report.

Chr11 data scope

This lab uses Salmon quantification files produced by the nf-core/rnaseq chr11 run in Lab 6. Only genes on chromosome 11 are quantified, so DESeq2 will test a subset of the full transcriptome. The pipeline steps, samplesheet format, contrasts file, and output structure are identical to a full-genome run — the chr11 scope simply reduces the number of tested genes and significant DEGs. For a full-genome analysis, replace --gtf with --genome GRCh38.p14.

When to Use nf-core/differentialabundance vs. Manual DESeq2

Both approaches call DESeq2 under the hood and will produce equivalent statistical results. Choose based on your needs:

Criterionnf-core/differentialabundanceManual DESeq2 (Jupyter/R)
ReproducibilityPipeline version-locked, containerizedDepends on your environment management
FlexibilityConfigured via parametersFull scripting flexibility
ReportingAutomated interactive Shiny report + MultiQCCustom plots and tables
Multiple contrastsDefined declaratively in a CSV fileManual loop or separate scripts
Best forStandard comparisons, sharing with collaborators, auditable analysesCustom filtering, novel visualizations, exploratory work

Part 2: Prepare Input Files

The nf-core/differentialabundance pipeline requires three input files:

  1. Sample metadata — a CSV describing each sample and its experimental condition (samplesheet.csv)
  2. Contrasts file — a CSV defining the statistical comparisons to perform (contrasts.csv)
  3. Quantification data — Salmon quant.sf output files from nf-core/rnaseq (already on disk)

A. Sample Metadata — samplesheet.csv

The samplesheet must have at minimum a sample column (matching the directory names inside the Salmon results folder) and one or more condition columns used for contrasts. Additional columns such as replicate are optional but recommended for clarity.

Create the samplesheet

cd /ibex/user/$USER/workshop/nfcore_da

cat > samplesheet.csv <<'EOF'
sample,condition,replicate
KO_1,TDP43_KD,1
KO_2,TDP43_KD,2
KO_3,TDP43_KD,3
WT_1,control,1
WT_2,control,2
WT_3,control,3
EOF

# Verify the file
cat samplesheet.csv
Adapting this samplesheet to your own study

The sample IDs in the sample column must exactly match the subdirectory names produced by nf-core/rnaseq inside the Salmon results folder (e.g., results/star_salmon/KO_1/). Always verify these names against the GEO page metadata for your dataset. For GSE136366, the KO_* samples are TARDBP/TDP-43 siRNA knockdowns and the WT_* samples are scramble-control replicates.

B. Contrasts File — contrasts.csv

The contrasts file tells the pipeline which comparisons to perform. Each row defines one contrast.

Create the contrasts file

cat > contrasts.csv <<'EOF'
id,variable,reference,target,blocking
TDP43_KD_vs_control,condition,control,TDP43_KD,
EOF

# Verify the file
cat contrasts.csv
Contrasts file column definitions
ColumnDescriptionExample
idA unique name for this comparison. Used to name output files and directories.TDP43_KD_vs_control
variableThe column in samplesheet.csv that defines the grouping.condition
referenceThe baseline group (denominator in the fold-change calculation).control
targetThe comparison group (numerator). Positive log2FC means higher in this group.TDP43_KD
blockingOptional. A comma-separated list of confounding variables to include in the model formula (e.g., batch effects).(empty)

C. Salmon Quantification Files

The Salmon quantification output from nf-core/rnaseq is already available. Each sample has its own subdirectory containing a quant.sf file.

Verify the Salmon output files

# Check the Salmon results directory from nf-core/rnaseq
ls /ibex/user/$USER/workshop/nfcore_rnaseq/results/star_salmon/

# Confirm each sample has a quant.sf file
ls /ibex/user/$USER/workshop/nfcore_rnaseq/results/star_salmon/KO_1/

# Quick sanity check on one quant.sf file
head -5 /ibex/user/$USER/workshop/nfcore_rnaseq/results/star_salmon/KO_1/quant.sf
What is in quant.sf?

Each quant.sf file contains five columns: Name (transcript ID), Length, EffectiveLength, TPM (transcripts per million), and NumReads (estimated read counts). The pipeline uses these estimated counts as input to DESeq2 via the tximeta R package, which correctly accounts for differences in effective transcript lengths.

Part 3: Run nf-core/differentialabundance

With all input files prepared, you can now launch the pipeline. For this lab, two options are provided: an interactive test run to verify the command, and a SLURM batch job for the full analysis.

Full pipeline run command

cd /ibex/user/$USER/workshop/nfcore_da

nextflow run nf-core/differentialabundance -r 1.5.0 \
    --input samplesheet.csv \
    --contrasts contrasts.csv \
    --matrix /ibex/user/$USER/workshop/nfcore_rnaseq/results/star_salmon/salmon.merged.gene_counts_length_scaled.tsv \
    --gtf /biocorelab/BIX/resources/genomes/workshops/human-chr11/GRCh38.chr11.gtf \
    --outdir da_results \
    --deseq2_vst_nsub 500 \
    -profile kaust \
    -resume
Key parameters explained
ParameterDescription
--inputPath to the sample metadata CSV (samplesheet.csv)
--contrastsPath to the contrasts CSV (contrasts.csv)
--matrixPath to the directory containing Salmon output subdirectories (one per sample)
--input_typeType of quantification input: salmon for Salmon quant directories, or featurecounts for a count matrix TSV
--gtfPath to the GTF annotation file; used for transcript-to-gene ID mapping. For chr11 lab we pass the chr11 GTF explicitly instead of using the KAUST genome registry (--genome GRCh38.p14).
--deseq2_vst_nsubNumber of genes sampled to fit the variance-stabilizing transformation (VST). Reduce for speed on small datasets.
-profile kaustActivates the KAUST institutional config: SLURM scheduler, Singularity containers, shared image library, and Ibex resource defaults
-resumeResume a previous run using cached intermediate results — avoids recomputing completed steps

Submit as a SLURM batch job (recommended for full datasets)

cd /ibex/user/$USER/workshop/nfcore_da

sbatch --job-name=nfcore_da \
    --mem=8G \
    --cpus-per-task=2 \
    --time=4:00:00 \
    --output=da_nextflow_%j.log \
    --wrap="module purge && module load nextflow && module load singularity && \
        export NXF_SINGULARITY_CACHEDIR=/ibex/user/$USER/singularity_cache && \
        nextflow run nf-core/differentialabundance -r 1.5.0 \
            --input samplesheet.csv \
            --contrasts contrasts.csv \
            --matrix /ibex/user/$USER/workshop/nfcore_rnaseq/results/star_salmon/salmon.merged.gene_counts_length_scaled.tsv \
            --gtf /biocorelab/BIX/resources/genomes/workshops/human-chr11/GRCh38.chr11.gtf \
            --outdir da_results \
            --deseq2_vst_nsub 500 \
            -profile kaust \
            -resume"
Monitoring the job

Track job progress with squeue -u $USER. Nextflow also writes a real-time log — tail it with:

Each completed process is marked with a green checkmark in the log. A .nextflow/ hidden directory in your working folder stores the execution cache used by -resume.

Understand the DESeq2 results table columns

# View column headers of the full results table
head -1 da_results/deseq2/TDP43_KD_vs_control/all_differential_expression.tsv | tr '\t' '\n' | cat -n
DESeq2 output columns
ColumnDescription
gene_idEnsembl gene identifier (e.g., ENSG00000000003)
gene_nameHGNC gene symbol (e.g., TARDBP)
baseMeanMean of normalized counts across all samples. Low values indicate low-abundance genes.
log2FoldChangeLog2 ratio of TDP43_KD over control. Positive = higher in knockdown.
lfcSEStandard error of the log2 fold-change estimate.
statWald test statistic.
pvalueNominal p-value from the Wald test.
padjBenjamini-Hochberg adjusted p-value (FDR). Use this for significance filtering.

Part 5: Interpret the Interactive Shiny Report

The shinyngs_app Shiny application provides an interactive interface for exploring differential expression results. It is a self-contained R app that can be run locally on your laptop after downloading it from Ibex.

What the Shiny app enables that static plots do not

Static PDF plots (like those in deseq2.plots.pdf) give you a fixed snapshot of your data. The Shiny app adds:

  • Interactive filtering — adjust log2FC and adjusted p-value thresholds in real time and see DEG counts update instantly
  • Gene-level drill-down — click any point on the volcano plot to see normalized count plots for that gene across all samples
  • Custom heatmaps — select subsets of genes and samples to visualize
  • Searchable DEG tables — sort and search the full results table without exporting to a spreadsheet

Sharing the Shiny report with collaborators

The shinyngs_app/ directory is a self-contained Shiny app — it includes the results data embedded in the app.R file. You can share it by:

  • Email / file transfer — zip the directory and send it; the recipient opens it in RStudio and clicks "Run App"
  • shinyapps.io — deploy for free with rsconnect::deployApp("shinyngs_app/") for a persistent URL
  • Posit Connect / internal server — deploy on your institution's Shiny Server for authenticated access

No bioinformatics installation is required on the collaborator's side — only R and the shiny package.

Navigate the key sections of the Shiny report

The Shiny app is organized into panels accessible from the top navigation bar:

  • Sample explorer — QC visualizations for all samples:
    • PCA plot: look for clear separation between control and TDP43_KD samples along PC1
    • Hierarchical clustering dendrogram: samples should cluster by condition
    • Sample-to-sample distance heatmap: identify outlier samples
  • Differential analysis — results for each contrast:
    • Volcano plot: x-axis = log2FC, y-axis = -log10(padj). Genes in the upper corners are significant and strongly changed.
    • MA plot: x-axis = mean expression (baseMean), y-axis = log2FC. Useful for detecting expression-level-dependent bias.
    • DEG table: sortable and searchable. Filter by padj and log2FC using the sliders.
  • Gene exploration — per-gene count plots:
    • Search for any gene by name (e.g., TARDBP) to see normalized counts per sample
    • Heatmap view: visualize expression patterns across selected gene sets

Why results might differ slightly from Lab 7

Both analyses use DESeq2, so results should be highly consistent. Minor differences may arise from: (1) the pipeline using tximeta to import Salmon counts with length correction, while Lab 7 may have used a plain count matrix; (2) different default filtering thresholds for low-count genes; or (3) different DESeq2 parameter defaults such as independentFiltering. These differences are expected and do not indicate an error in either approach.

Part 6: Interpret the MultiQC Report

The pipeline generates a MultiQC report that aggregates QC metrics and DESeq2 summary statistics into a single HTML file.

Open the MultiQC report

# Download the MultiQC report to your laptop
scp username@ilogin.ibex.kaust.edu.sa:/ibex/user/$USER/workshop/nfcore_da/da_results/multiqc/multiqc_report.html ./

# Open in your browser
open multiqc_report.html    # macOS
# xdg-open multiqc_report.html   # Linux

Key sections to review in the MultiQC report

Navigate through the report sections and look for the following:

  • DESeq2 — Dispersion estimates: The dispersion plot shows how per-gene variance estimates were shrunk toward the fitted mean-dispersion trend. Well-behaved data shows a smooth trend line with most gene-level estimates clustering closely around it.
  • DESeq2 — PCA: The PCA plot in MultiQC matches the one in the Shiny app. Confirm PC1 separates control from TDP43_KD samples — this is the primary axis of variation in a well-designed knockdown experiment.
  • DESeq2 — MA plot: In the MA plot, most genes should lie along the horizontal axis (no fold change). Significantly DE genes are highlighted in color. A symmetric distribution of up- and down-regulated genes is typical; a strong asymmetry may indicate a normalization issue or a large-scale transcriptional response to TDP-43 loss.
  • DESeq2 — Summary statistics: A table summarizing total genes tested, number significant at the default thresholds, and the up/down breakdown for each contrast.
