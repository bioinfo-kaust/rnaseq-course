Lab 5: Quantification with Salmon & Exploratory Analysis

In this lab you will quantify gene expression for all GSE136366 samples using Salmon quasi-mapping, then import the results into R, apply variance-stabilising transformation (VST), and produce a PCA plot and sample-distance heatmap to assess sample quality.

Using subsampled data in this lab

The input files (trimmed/${SRR}_trimmed_1/2.fastq.gz) are the trimmed, subsampled reads produced in Labs 2 and 3 — each sample contains 1 million read pairs. Salmon finishes quantifying a single subsampled sample in under 1 minute. All commands are identical to a full-scale analysis; only the data size differs. Expect lower per-gene counts than a full dataset, which is expected and appropriate for a workshop exercise.

Learning Objectives
  • Build a Salmon transcriptome index from the Ensembl cDNA FASTA
  • Run Salmon quasi-mapping quantification on all GSE136366 paired-end samples
  • Import Salmon output into R using tximport (transcript → gene summarisation)
  • Apply variance-stabilising transformation (VST) with DESeq2
  • Produce a PCA plot and sample-distance heatmap to evaluate sample quality and clustering

Part 2: Build a Salmon Transcriptome Index

Salmon performs quasi-mapping against a transcriptome FASTA (not the genome). This makes it much faster than alignment-based methods such as STAR, while still producing accurate quantification estimates.

Why a Transcriptome FASTA?

Salmon does not align reads to the genome. Instead it uses a k-mer index built from all annotated transcript sequences. Reads are matched against this index in memory, allowing Salmon to process a typical RNA-seq sample in a few minutes rather than hours. Because it works at transcript level, Salmon natively handles multi-mapping reads and produces both transcript-level and gene-level estimates.

Download the Ensembl human transcriptome (cDNA)

cd /ibex/user/$USER/workshop

# Download Ensembl release 110 cDNA sequences for Homo sapiens
wget https://ftp.ensembl.org/pub/release-110/fasta/homo_sapiens/cdna/Homo_sapiens.GRCh38.cdna.all.fa.gz

# Move to the references directory with a cleaner name
mv Homo_sapiens.GRCh38.cdna.all.fa.gz references/GRCh38_transcriptome.fa.gz

# Verify the file is present
ls -lh references/
Ensembl cDNA vs genome FASTA

The cDNA file contains spliced transcript sequences (one entry per annotated transcript isoform). It is much smaller than the full genome (~200 MB vs ~3 GB) and is exactly what Salmon needs to build its index.

Build the Salmon index

salmon index \
    -t references/GRCh38_transcriptome.fa.gz \
    -i references/salmon_index \
    --threads 8

This takes roughly 5–10 minutes and only needs to be done once. The resulting references/salmon_index/ directory can be reused for all samples and future datasets mapped to the same annotation release.

Part 3: Quantify with Salmon

Now that the index is ready, run Salmon on each trimmed sample. We will loop over all sample IDs listed in raw_data/SRR_Acc_List.txt.

Key Salmon Parameters
ParameterMeaning
-iPath to the Salmon index directory
-l AAuto-detect library/strand type (recommended for most datasets)
-1 / -2R1 and R2 FASTQ files (paired-end)
--validateMappingsMore accurate mapping using alignment-based validation; recommended
--gcBiasCorrect for GC-content bias in fragment sequencing
--seqBiasCorrect for sequence-specific biases at read ends
--threadsNumber of CPU threads to use
-oOutput directory for this sample

Run Salmon quantification for all samples

cd /ibex/user/$USER/workshop

mkdir -p counts/salmon

for srr in $(cat raw_data/SRR_Acc_List.txt); do
    echo "=== Quantifying $srr ==="
    salmon quant \
        -i references/salmon_index \
        -l A \
        -1 trimmed/${srr}_trimmed_1.fastq.gz \
        -2 trimmed/${srr}_trimmed_2.fastq.gz \
        --validateMappings \
        --gcBias \
        --seqBias \
        --threads 8 \
        -o counts/salmon/${srr}
    echo "Done: $srr"
done
Expected runtime

Each subsampled sample (1 M read pairs) takes well under 1 minute. The full loop for all GSE136366 samples should complete in a few minutes within your interactive session.

What Does the Strandedness Detection Tell You?

The lib_format_counts.json file reports how many reads matched each possible library orientation. A high count for SF or SR indicates a stranded library; roughly equal counts for IU (inward, unstranded) indicate an unstranded library. For GSE136366, expect an unstranded paired-end library (IU).

Create a transcript-to-gene mapping from the GTF

Salmon quantifies at the transcript level. We need a mapping table to aggregate transcript counts to gene-level counts. GenomicFeatures can build this directly from the GTF file:

# Build a TxDb object from the GTF annotation
txdb <- makeTxDbFromGFF("references/GRCh38_genes.gtf", format = "gtf")

# Extract the transcript-to-gene mapping
k <- keys(txdb, keytype = "TXNAME")
tx2gene <- select(txdb, k, "GENEID", "TXNAME")

# Inspect the mapping table
head(tx2gene)
dim(tx2gene)  # should have one row per transcript
tx2gene Format

The tx2gene data frame must have two columns: the first contains transcript IDs (matching the Name column in quant.sf) and the second contains gene IDs. tximport uses this table to sum transcript-level estimates up to gene level.

Import Salmon counts with tximport

# Read the list of SRR accessions
srr_ids <- readLines("raw_data/SRR_Acc_List.txt")
srr_ids  # preview

# Build paths to each quant.sf file
quant_files <- file.path("counts/salmon", srr_ids, "quant.sf")
names(quant_files) <- srr_ids

# Verify all files exist before importing
stopifnot(all(file.exists(quant_files)))

# Import — tximport sums transcript counts to gene level
txi <- tximport(quant_files, type = "salmon", tx2gene = tx2gene)

# Explore the result
names(txi)          # components: counts, abundance (TPM), length
dim(txi$counts)     # rows = genes, columns = samples
head(txi$counts)
TPM vs Raw Counts — Which to Use?

TPM (Transcripts Per Million) normalises for transcript length and sequencing depth. It is useful for comparing expression levels within a sample or ranking genes by abundance. However, TPM values are compositional (they always sum to 1 million per sample), so they are not appropriate for differential expression testing.

Raw counts (estimated read counts from Salmon) retain the variance structure expected by count-based statistical models such as DESeq2 and edgeR. Always use raw counts for differential expression analysis.

tximport imports both: txi$abundance = TPM, txi$counts = estimated counts.

Build a DESeq2 object and apply VST

# Create DESeqDataSet from tximport output
dds <- DESeqDataSetFromTximport(txi, colData = coldata, design = ~condition)

# Filter out genes with very low counts (speeds up analysis and improves statistics)
dds <- dds[rowSums(counts(dds)) >= 10, ]
cat("Genes retained after filtering:", nrow(dds), "\n")

# Apply variance-stabilising transformation (VST)
# blind = TRUE: transformation ignores the experimental design — appropriate for QC/exploration
vsd <- vst(dds, blind = TRUE)

# Inspect the transformed data
assay(vsd)[1:5, 1:3]
Why VST Before PCA?

Raw RNA-seq count data have a mean-variance relationship: highly expressed genes have much larger variance than lowly expressed ones. If you run PCA directly on raw counts, the first principal components will be dominated by a handful of highly expressed genes rather than reflecting true sample-level differences.

Variance-stabilising transformation (VST) removes this mean-variance dependence, giving each gene roughly equal weight in the PCA. This makes the resulting plots far more interpretable. An alternative is rlog(), which is more robust for very small sample sizes (< 15 samples) but slower to compute.

PCA plot

# Calculate PCA and retrieve data for ggplot2
pca_data    <- plotPCA(vsd, intgroup = "condition", returnData = TRUE)
percent_var <- round(100 * attr(pca_data, "percentVar"))

# Plot with ggplot2
ggplot(pca_data, aes(PC1, PC2, color = condition, label = name)) +
    geom_point(size = 4) +
    geom_text(vjust = -0.8, size = 3) +
    xlab(paste0("PC1: ", percent_var[1], "% variance")) +
    ylab(paste0("PC2: ", percent_var[2], "% variance")) +
    theme_bw() +
    ggtitle("PCA \u2014 GSE136366 Samples (VST)")

# Save to file
ggsave("results/pca_plot.pdf", width = 7, height = 5)
Interpreting the PCA Plot

A well-behaved RNA-seq experiment should show:

  • Replicates from the same condition clustering tightly together
  • The two conditions (control vs knockdown) separating along PC1 (the axis of greatest variation)

If replicates from the same condition are far apart, this may indicate a sample swap, batch effect, or poor data quality for that sample. If conditions do not separate at all, the perturbation (TDP-43 knockdown) may have a modest transcriptional effect, or there may be a problem with the experimental conditions.

Sample-distance heatmap

# Compute pairwise Euclidean distances between samples (on VST data)
sampleDists       <- dist(t(assay(vsd)))
sampleDistMatrix  <- as.matrix(sampleDists)

# Plot as a heatmap
pheatmap(sampleDistMatrix,
    clustering_distance_rows = sampleDists,
    clustering_distance_cols = sampleDists,
    main     = "Sample-to-sample distances (VST)",
    filename = "results/sample_distance_heatmap.pdf")

# Also display in interactive R session
pheatmap(sampleDistMatrix,
    clustering_distance_rows = sampleDists,
    clustering_distance_cols = sampleDists,
    main = "Sample-to-sample distances (VST)")
Interpreting the Sample-Distance Heatmap

The heatmap shows pairwise distances between all samples. Darker blue cells indicate more similar samples (smaller distance). You should see:

  • Replicates of the same condition clustering together (a block of dark-blue cells along the diagonal)
  • Between-condition pairs appearing as lighter cells (greater distance)

Any sample that clusters with the wrong condition group is a warning sign that warrants investigation before proceeding to differential expression analysis.

Exercise 2: Salmon Mapping Rate

What is the mapping rate for each sample? Salmon reports this in its run log and in the auxiliary output directory.

Hint: Check counts/salmon/<SRR>/aux_info/meta_info.json for the percent_mapped field. A healthy mapping rate for a human RNA-seq sample against a well-annotated transcriptome is typically 70–90%. Rates below 50% may indicate contamination, adapter content, wrong reference, or non-coding RNA libraries. You can also check the SLURM log for the line Mapping rate = ... printed by Salmon at the end of each run.

Exercise 3: PCA Sample Clustering

Do the GSE136366 samples cluster by condition (control vs knockdown) along PC1 in your PCA plot? What biological or technical factors could explain it if they do not?

Consider: batch effects, library size differences, outlier samples, or a weak transcriptional response to the TDP-43 knockdown. If you see an unexpected outlier, check whether its mapping rate or total count differs markedly from the other samples.

Exercise 4: Highest Average TPM Gene

Which gene has the highest average TPM across all samples? TPM values are stored in txi$abundance.

Hint: Calculate row means with rowMeans(txi$abundance), then use which.max() to find the index of the maximum value, and retrieve the gene name. Highly expressed genes are often housekeeping genes such as ACTB (beta-actin) or ribosomal protein genes.
