In this lab, you will use DESeq2 to identify differentially expressed genes (DEGs) in the GSE136366 dataset — an RNA-seq study of TDP-43 (TARDBP) knockdown in human cells. You will import Salmon quantification counts, build a statistical model, run DE analysis with log fold-change shrinkage, and produce publication-quality visualizations.

Dataset Background: GSE136366

GSE136366 is a human RNA-seq dataset studying the effect of TDP-43 (encoded by TARDBP) knockdown. TDP-43 is an RNA-binding protein involved in RNA splicing and stability, and is implicated in ALS and frontotemporal dementia. The experimental design includes control samples and TDP-43 knockdown (KD) samples, each with biological replicates.

Part 2: Import Counts and Build DESeq2 Object

DESeq2 requires raw, un-normalized counts as input. We will use tximport to import Salmon's quantification output, which correctly accounts for transcript length differences across samples (important when using count-based methods like DESeq2).

Chr11 data scope

In this lab the Salmon counts come from nf-core/rnaseq run on chr11-filtered reads (Lab 6). Only genes on chromosome 11 are quantified. The DESeq2 workflow, contrasts, and visualizations are identical to a full-genome analysis — the chr11 scope simply means the number of tested genes and significant DEGs will be smaller.

Build a transcript-to-gene mapping (tx2gene)

Salmon quantifies at the transcript level. We need a mapping table from transcript IDs to gene IDs to aggregate counts to the gene level. We will build this directly from the chr11 GTF file.

# Build transcript-to-gene mapping from the chr11 GTF
library(GenomicFeatures)
gtf_path <- "/biocorelab/BIX/resources/genomes/workshops/references/chr11/GRCh38.chr11.gtf"

library(txdbmaker)
txdb <- makeTxDbFromGFF(gtf_path, format = "gtf")

k       <- keys(txdb, keytype = "TXNAME")
tx2gene <- select(txdb, k, "GENEID", "TXNAME")

head(tx2gene)
cat("Transcript-to-gene mappings:", nrow(tx2gene), "\n")

What does tximport do?

tximport aggregates transcript-level estimates to the gene level. Importantly, it imports both the estimated counts and the transcript length offsets (average transcript length per gene per sample). When these are passed to DESeq2, it adjusts for differences in isoform usage across samples — this is more accurate than simply summing raw counts.

Create sample metadata (coldata)

GSE136366 contains control samples (WT_*) and TDP-43 knockdown (KO_*) samples. The sample IDs encode the condition, making it straightforward to define the experimental design.

# GSE136366: KO_* samples are TDP-43 knockdown, WT_* are controls
sample_ids <- c(
    "KO_1", "KO_2", "KO_3",
    "WT_1", "WT_2", "WT_3"
)

condition <- ifelse(grepl("^KO", sample_ids), "TDP43_KD", "control")

coldata <- data.frame(
    sample    = sample_ids,
    condition = factor(condition, levels = c("control", "TDP43_KD")),
    row.names = sample_ids
)

coldata$condition <- relevel(coldata$condition, ref = "control")

# Verify
coldata
levels(coldata$condition)   # Should show: "control" "TDP43_KD"

The order of samples in coldata must exactly match the order of columns in the txi$counts matrix. Check this with all(rownames(coldata) == colnames(txi$counts)) — it must return TRUE before proceeding. Since both quant_files and coldata are built from the same sample_ids vector, the order is guaranteed to match.

# This must return TRUE

Build the DESeq2 object and pre-filter

# Create DESeqDataSet from tximport object
dds <- DESeqDataSetFromTximport(txi, colData=coldata, design=~condition)

# Inspect the object
dds

# Pre-filter: remove genes with very low counts across all samples
# Keep genes with at least 10 counts in at least 3 samples
keep <- rowSums(counts(dds) >= 10) >= 3
dds  <- dds[keep, ]

cat("Genes retained after filtering:", nrow(dds), "\n")
Why Pre-filter?

Genes with very few reads across all samples carry no statistical power and inflate the multiple testing burden. Removing them speeds up the analysis and improves the sensitivity of adjusted p-value calculations. A common threshold is: keep genes where at least N samples have a count of 10 or more, where N equals the size of the smallest group (here, 3 replicates).

Part 3: Run DESeq2

DESeq2 fits a negative binomial generalized linear model to estimate gene-wise dispersion and tests for differential expression using the Wald test.

Run the DESeq2 analysis

# Run the full DESeq2 pipeline (normalization, dispersion estimation, model fitting)
dds <- DESeq(dds)

# Check size factors — these represent relative sequencing depth per sample
# Values close to 1.0 indicate similar library sizes
normalizationFactors(dds)

# View the available results names (contrasts defined by the model)
resultsNames(dds)

Inspect dispersion estimates

# Plot dispersion estimates — useful quality check
pdf("dispersion_estimates.pdf")
plotDispEsts(dds, main="Dispersion Estimates: GSE136366")
dev.off()
Reading the Dispersion Plot

The dispersion plot shows gene-wise dispersion estimates (black dots), the fitted mean-dispersion trend (red line), and the final shrunken estimates (blue dots). In a well-behaved dataset you should see dispersion decrease as mean expression increases, and the shrunken estimates clustering around the trend line. Outlier genes with very high dispersion will appear as isolated black dots above the trend.

Extract results

# Extract DE results for the contrast: TDP43_KD vs. control
res <- results(dds, contrast=c("condition", "TDP43_KD", "control"))

# Print a summary: total significant genes at default alpha=0.1
summary(res)

# View the top results sorted by adjusted p-value
head(res[order(res$padj), ])
Understanding the p-value vs. Adjusted p-value

In a typical RNA-seq experiment, DESeq2 tests tens of thousands of genes simultaneously. If we used the raw p-value (threshold 0.05), we would expect 5% of all tests to be false positives by chance alone — that could mean thousands of spurious hits. The adjusted p-value (also called the FDR, or Benjamini-Hochberg corrected p-value) accounts for this multiple testing problem by controlling the expected proportion of false discoveries among all significant results. Always use padj, not pvalue, for filtering significant genes.

Apply LFC shrinkage with apeglm

# Apply log fold-change shrinkage using the apeglm method
# coef must match a name from resultsNames(dds)
res_shrunk <- lfcShrink(
    dds,
    coef = "condition_TDP43_KD_vs_control",
    type = "apeglm"
)

# Summary of shrunken results
summary(res_shrunk)
Why Apply LFC Shrinkage?

For genes with very low counts, the raw log2 fold-change estimates are noisy and often extreme (e.g., LFC = 8 for a gene with 1 count in one group and 3 in another). LFC shrinkage pulls these unreliable estimates toward zero, weighted by the amount of information (expression level and dispersion) available for each gene. This produces more accurate and interpretable fold-change values, particularly for ranking and visualization. The apeglm method (Adaptive t Prior shrinkage) is the recommended default in current DESeq2 versions.

Note: shrinkage does not change p-values or which genes are called significant — only the fold-change estimates are modified.

Export results to a table

Recommended Thresholds for Filtering DEGs

There is no universal threshold, but a widely used convention in RNA-seq studies is:

  • Adjusted p-value: padj < 0.05 — controls the false discovery rate at 5%
  • Log2 fold-change: |log2FoldChange| > 1 — corresponds to a 2-fold change, filtering for biologically meaningful effect sizes

More stringent studies use padj < 0.01 and |LFC| > 2. Always report the exact thresholds used in your methods section. Note that the LFC threshold is separate from the statistical significance threshold — a gene can be statistically significant but biologically uninteresting if its fold-change is very small.

Part 4: Visualizations

Visualizations help communicate the scale and pattern of differential expression. We will produce three standard plots: an MA plot, a volcano plot, and a heatmap of the top DEGs.

MA Plot

The MA plot shows the relationship between mean expression (x-axis) and log2 fold-change (y-axis). It helps assess whether the LFC shrinkage worked correctly and whether there is any systematic bias.

# MA plot using the shrunken LFC estimates
# Points in blue are significant at padj < 0.05
png("results/MA_plot_TDP43_KD_vs_Control.png", width = 800, height = 600, res = 120)
plotMA(res_shrunk,
    ylim = c(-5, 5),
    main = "MA Plot: TDP-43 KD vs. Control")
dev.off()
Interpreting the MA Plot

In a well-normalized experiment, the bulk of genes (grey dots) should cluster around LFC = 0 across all expression levels. Colored points are significant hits. LFC shrinkage ensures that low-expression genes (left side of the plot) no longer show extreme fold-change values — without shrinkage, low-count genes scatter wildly along the y-axis.

Volcano Plot

The volcano plot combines statistical significance and fold-change magnitude, allowing rapid identification of the most strongly dysregulated genes.

# Classify genes by significance and direction
res_df <- res_df %>%
    mutate(
        significance = case_when(
            padj < 0.05 & log2FoldChange >  1 ~ "Up",
            padj < 0.05 & log2FoldChange < -1 ~ "Down",
            TRUE                               ~ "NS"
        )
    )

# Count genes in each category
table(res_df$significance)

# Build the volcano plot
ggplot(res_df %>% filter(!is.na(padj)),
       aes(x=log2FoldChange, y=-log10(padj))) +
    geom_point(aes(color=significance), alpha=0.5, size=1) +
    scale_color_manual(
        values = c("Up"="#D62728", "Down"="#1F77B4", "NS"="grey60")
    ) +
    geom_vline(xintercept=c(-1, 1), linetype="dashed", color="grey40") +
    geom_hline(yintercept=-log10(0.05), linetype="dashed", color="grey40") +
    geom_text_repel(
        data    = res_df %>% filter(padj < 0.001 & abs(log2FoldChange) > 2) %>% head(20),
        aes(label=gene_id),
        size    = 2.5,
        max.overlaps = 20
    ) +
    labs(
        x     = "Log2 Fold Change",
        y     = "-log10(adjusted p-value)",
        title = "Volcano Plot: TDP-43 KD vs. Control",
        color = "Regulation"
    ) +
    theme_bw()

# Save the plot
ggsave("results/volcano_plot.pdf", width=8, height=6)
cat("Volcano plot saved to results/volcano_plot.pdf\n")

Heatmap of Top 50 DEGs

A heatmap of the most significantly differentially expressed genes shows expression patterns across all samples and whether replicates cluster together as expected.

# Select the top 50 significant DEGs by adjusted p-value
top50 <- res_df %>%
    filter(padj < 0.05, !is.na(padj)) %>%
    slice_min(padj, n=50) %>%
    pull(gene_id)

cat("Number of genes selected for heatmap:", length(top50), "\n")

# Apply variance-stabilizing transformation (VST) for visualization
# blind=FALSE uses the design matrix — preferred when groups differ
vsd <- varianceStabilizingTransformation(dds, blind=FALSE)

# Extract the VST-normalized count matrix for top 50 genes
mat <- assay(vsd)[top50, ]

# Center each gene (subtract row mean) to highlight relative changes
mat <- mat - rowMeans(mat)

# Build annotation for columns (samples)
annotation_col <- data.frame(
    condition = coldata$condition,
    row.names = rownames(coldata)
)

# Draw the heatmap and save as PDF
pheatmap(mat,
    annotation_col = annotation_col,
    cluster_rows   = TRUE,
    cluster_cols   = TRUE,
    show_rownames  = TRUE,
    fontsize_row   = 7,
    main           = "Top 50 DEGs: TDP-43 KD vs. Control",
    filename       = "results/heatmap_top50DEGs.pdf"
)

cat("Heatmap saved to results/heatmap_top50DEGs.pdf\n")
VST vs. rlog Normalization

For visualization purposes, raw counts are not suitable because highly expressed genes dominate the color scale. DESeq2 provides two variance-stabilizing transformations:

  • VST (vst()) — Fast, recommended for datasets with more than ~30 samples or for quick exploratory analysis.
  • rlog (rlog()) — Slower but may perform better for small datasets (< 30 samples) with very different library sizes.

Neither transformation is used for the actual DE testing — DESeq2 uses raw counts internally for that. VST/rlog are only for visualization and PCA.
